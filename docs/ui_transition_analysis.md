# UI 전환 분석: Desktop (PyQt6) vs Web (React/Vue)

현재 **PyQt6 기반의 데스크탑 애플리케이션**을 **웹 기반 인터페이스**로 전환할 경우의 장단점을 분석한 문서입니다.

## 1. 장점 (Pros)

### 1.1 접근성 및 모바일 지원 (Accessibility)
- **어디서든 접속 가능**: 집 밖에서도 스마트폰이나 태블릿으로 매매 현황을 확인하고 긴급 정지(Panic Button)를 누를 수 있습니다.
- **다중 기기 지원**: 여러 모니터나 여러 컴퓨터에서 동시에 대시보드를 띄워놓을 수 있습니다.

### 1.2 현대적인 UI/UX (Modern Design)
- **디자인 자유도**: CSS와 최신 웹 프레임워크(React, TailwindCSS)를 사용하여 훨씬 세련되고 트렌디한 디자인(애니메이션, 그라디언트 등)을 구현하기 쉽습니다.
- **강력한 차트 라이브러리**: TradingView의 Lightweight Charts 등 웹 생태계의 고성능 차트 라이브러리를 활용할 수 있습니다.

### 1.3 배포 및 업데이트 (Deployment)
- **클라이언트 무설치**: 브라우저만 있으면 되므로, UI 업데이트 시 사용자가 프로그램을 다시 설치할 필요 없이 새로고침만 하면 됩니다.

---

## 2. 단점 (Cons)

### 2.1 아키텍처 복잡도 증가 (Complexity)
- **Client-Server 구조 필요**:
    - 현재: `UI <-> Core`가 메모리를 공유하며 직접 통신 (단일 프로세스).
    - 웹: `Web UI <-> API Server (FastAPI) <-> Core` 구조로 변경 필요.
- **개발 공수**: 백엔드 API(REST/WebSocket)를 새로 개발해야 하며, 프론트엔드 코드를 처음부터 다시 작성해야 합니다.

### 2.2 키움 API의 제약 (Kiwoom Dependency)
- **Windows 종속성**: 키움 API(OCX)는 Windows 환경에서만 동작합니다. 따라서 웹으로 바꾸더라도 **백엔드 서버는 반드시 Windows PC에서 실행**되어야 합니다. (클라우드 리눅스 서버 단독 호스팅 불가)

### 2.3 실시간 성능 및 지연 (Latency)
- **네트워크 오버헤드**: 로컬호스트라 하더라도 메모리 직접 참조보다 통신 비용이 발생합니다.
- **고빈도 데이터 처리**: 호가(Orderbook)나 틱 데이터가 초당 수십/수백 건 들어올 때, 브라우저 렌더링 최적화가 PyQt보다 까다로울 수 있습니다.

---

## 3. 기술적 과제 (Technical Challenges)

| 구분 | Desktop (Current) | Web (Proposed) |
| :--- | :--- | :--- |
| **통신 방식** | Python 함수 호출 / Signal & Slot | REST API (요청/응답) + WebSocket (실시간) |
| **데이터 공유** | 메모리 객체 공유 | JSON 직렬화/역직렬화 필요 |
| **보안** | 로컬 실행으로 보안 이슈 적음 | 외부 접속 허용 시 인증/보안(HTTPS, JWT) 필수 |
| **개발 언어** | Python 100% | Python (Backend) + JS/TS (Frontend) |

---

## 4. 종합 추천 (Recommendation)

### 시나리오 A: "집 밖에서도 매매 상황을 보고 제어하고 싶다"
👉 **Web 전환 추천 (또는 하이브리드)**
- 기존 PyQt는 유지하되, 모니터링 및 간단한 제어(매매 중지 등)만 가능한 **경량 웹 대시보드**를 추가로 개발하는 방식을 추천합니다. (FastAPI로 데이터만 서빙)

### 시나리오 B: "단지 UI가 더 예뻤으면 좋겠다"
👉 **Web 전환 비추천**
- 배보다 배꼽이 더 큽니다. PyQt에서도 QSS(Qt Style Sheets)와 커스텀 위젯을 통해 충분히 미려한 UI를 만들 수 있습니다.

### 시나리오 C: "상용 서비스로 배포하고 싶다"
👉 **Web 전환 필수**
- 다수의 사용자에게 배포하려면 웹 기반이 유지보수 및 확장에 유리합니다. 단, 각 사용자의 PC에 '매매 엔진(Core)'을 설치하고, 웹은 그 엔진과 통신하는 인터페이스 역할만 해야 합니다.
