# Phase 4: 설정 및 관리 - 심화 고려사항 (Advanced Considerations)

## 1. 안전성 (Safety & Integrity)

### 1.1 핫 스왑 방지 (Hot-Swap Protection)
- **문제**: 전략이 실행 중일 때 파라미터(예: 레버리지, 타임프레임)를 변경하면 내부 상태와 불일치가 발생하여 오작동할 수 있습니다.
- **해결**:
    - 전략 실행 중(`Running` 상태)에는 파라미터 수정 UI를 **비활성화(Disabled)** 처리합니다.
    - 또는, 수정 시도시 "전략을 중지하고 적용하시겠습니까?" 팝업을 띄우고, 승인 시 `Stop -> Apply -> Start` 과정을 자동화합니다.

### 1.2 설정 유효성 검사 (Validation)
- **범위 검사**: 입력 필드(`QSpinBox` 등)의 Min/Max 제한 외에도, 논리적 유효성을 검사해야 합니다. (예: 손절가가 목표가보다 높을 수 없음)
- **의존성 검사**: 특정 옵션 활성화 시 필수값 입력 여부 확인 (예: 카카오톡 알림 켜기 -> 토큰 값 필수).

---

## 2. 편의성 및 확장성 (Usability & Scalability)

### 2.1 설정 프로필 (Configuration Profiles)
- **기능**: 시장 상황에 따라 다른 설정값을 미리 저장해두고 불러오는 기능.
    - 예: "상승장_공격형.yaml", "횡보장_안전형.yaml"
- **구현**: 설정 창 하단에 `[프로필 저장]`, `[프로필 불러오기]` 버튼 추가.

### 2.2 데이터베이스 관리 (DB Maintenance)
- **문제**: 장기간 운용 시 `trade.db`에 틱 데이터나 로그가 쌓여 용량이 커지고 성능이 저하될 수 있습니다.
- **해결**:
    - **Auto Vacuum**: 설정 탭에 "DB 최적화" 버튼 추가.
    - **Data Pruning**: "30일 지난 데이터 삭제" 옵션 제공.

### 2.3 테마 및 접근성 (Theme & Accessibility)
- **폰트 크기**: 고해상도 모니터나 노안 사용자를 위해 UI 폰트 크기 조절 기능 (`Zoom In/Out`).
- **테마 선택**: 현재 Dark Mode 고정이지만, Light Mode 또는 High Contrast 모드 지원 고려.

---

## 3. 알림 및 리포트 고도화

### 3.1 알림 채널 다변화
- 현재 카카오톡만 지원하지만, 추후 **Telegram**, **Slack**, **Email** 등으로 확장할 수 있도록 인터페이스(`NotificationProvider`)를 설계해야 합니다.
- 설정 UI에서도 "알림 채널"을 선택하거나 복수 선택할 수 있게 확장성을 열어둡니다.

### 3.2 테스트 발송 기능
- 설정만 하고 실제 상황에서 알림이 안 오면 낭패입니다.
- 설정 창에 **[테스트 메시지 발송]** 버튼을 두어, 즉시 내 폰으로 알림이 오는지 확인할 수 있게 합니다.

## 4. 버전 관리 및 업데이트
- **버전 정보**: 설정 창 하단에 현재 버전(`v1.0.0`) 표시.
- **업데이트 확인**: GitHub API 등을 통해 최신 버전이 있는지 확인하고 알려주는 기능.
